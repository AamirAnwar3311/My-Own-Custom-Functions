# Contributing to HelperFunctions

First off, thank you for considering contributing to HelperFunctions! It's people like you that make HelperFunctions such a great package.

## Code of Conduct

- Be respectful and inclusive
- Be collaborative
- Be open to feedback
- Focus on what is best for the community
- Show empathy towards other community members

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:
- Clear title and description
- Steps to reproduce the bug
- Expected behavior
- Actual behavior
- PHP and Laravel versions
- Any relevant code examples

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include:
- Use case for the enhancement
- Current behavior
- Proposed behavior
- Any relevant examples

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/amazing-feature`
3. Make your changes
4. Ensure code follows PSR-12 coding standards
5. Add or update tests if applicable
6. Update documentation if needed
7. Commit your changes: `git commit -m 'Add amazing feature'`
8. Push to the branch: `git push origin feature/amazing-feature`
9. Submit a pull request

### Coding Standards

- Follow PSR-12 coding standards
- Add PHPDoc comments for all public methods
- Keep functions focused and single-purpose
- Add type hints and return types
- Write clear and descriptive commit messages

### Testing

- Add unit tests for new features
- Ensure all existing tests pass
- Aim for high test coverage

## Getting Started

1. Clone the repository
2. Run `composer install`
3. Run tests with `phpunit`

## Questions?

Feel free to open an issue for any questions or clarifications.
